# Start with F0

F0 is Factorial's design system — the components, patterns, tokens and rules every Factorial product shares. This is its home: where you come to **understand** it and to **contribute** to it.

<KeyPoint title="The door is your agent">
Whatever you're doing, start by talking to **your AI agent** in the [`factorialco/f0`](https://github.com/factorialco/f0) repo. F0 ships its rules as **AGENTS.md** and skills, so any AGENTS.md-aware agent — Claude Code, Codex, OpenCode, or whatever you use — reads them, knows the catalogue, and builds to the standard with you. When a real decision is needed, it happens in [#f0-support](https://factorialteam.slack.com/archives/C082ZNKS403).
</KeyPoint>

**Set it up once.** Open the [`factorialco/f0`](https://github.com/factorialco/f0) repo in your agent — it auto-loads the F0 skills. New to it? Ask it to clone and set the repo up for you; there's nothing manual to track.

**Not sure how to phrase it? Try:**

- _"Give me a quick tour of F0 — what's here and how is it organized?"_
- _"Does F0 already have a component for picking a date range? Show me how to use it."_
- _"I want to add a component that doesn't exist yet — walk me through contributing it."_

## What are you here to do?

### Understand F0

What the system is and how it's built. Start with **[What is F0](/docs/about-f0-what-is-f0--documentation)** — what it is, why it exists, and the five principles. Then the model:

- [Where it goes](/docs/about-f0-where-it-goes--documentation) — how F0 is organized: Core / Kits / Domain specific
- [Components Maturity](/docs/about-f0-components-maturity--documentation) — the three levels: experimental, stable, deprecated

### Contribute to F0

Add or change something in the system itself. Start with **[How to contribute](/docs/about-f0-how-to-contribute--documentation)** — the whole flow, from idea to stable. The rest of the contract, as you need it:

- [Definition of Done](/docs/about-f0-definition-of-done--documentation) — the bar a change must meet
- [Design Review](/docs/about-f0-design-review--documentation) — what a design review checks
- [Release &amp; Versioning](/docs/about-f0-release-and-versioning--documentation) — versioning and the deprecation policy

---

_Just building a screen day to day?_ That's **prototyping**, not contributing — describe the screen to your agent and it composes a real, working screen from existing F0 components in **F0 Compose**. You're using the system, not changing it.
