# How to add a new icon

Icons in F0 are defined once in `packages/core/assets/icons` as SVG files and then generated into framework-specific components (React, React Native, etc.).

## Directory structure

```
packages/core/assets/icons/
  app/        # General purpose icons
  modules/    # Factorial product module icons
  animated/   # Motion-enhanced icons
  ai/         # Factorial AI-related icons
```

Choose the directory that best matches the icon's purpose.

## SVG requirements

- Single color only (no hardcoded fills — use `currentColor` or stroke).
- Must fit in a square — 24×24px recommended.
- File name in **PascalCase** (e.g. `ChevronDown.svg`).

## Steps

1. Add the SVG file to the appropriate directory in `packages/core/assets/icons`.
2. Run the icon generation command in the target package:

```bash
# From packages/react
pnpm generate-icons
```

This generates React components into `packages/react/src/icons/`.

3. Verify the new icon appears in the [Icons browser](?path=/docs/foundations-icons--docs) and renders correctly.

## Animated icons

Animated icons support a `state` prop (`"normal"` | `"animate"`). The SVG must include the animation definition — refer to an existing animated icon in `packages/core/assets/icons/animated` as a reference.

## Module icons

After adding a module icon SVG, register it in `packages/react/src/components/avatars/F0AvatarModule/modules.ts` so it becomes available via `F0AvatarModule`.
