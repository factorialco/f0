# Colors

F0 exposes two sets of color tokens: semantic tokens and bare colors.

## Semantic Tokens

**Semantic tokens are the preferred way of setting colors.** Semantic tokens are
meaningful names used to describe the purpose or intent of a color, rather than
specifying a direct value. Instead of defining tokens based on their appearance
(e.g., blue, green, red), semantic tokens define elements based on their
function or intent in the UI. For example,
`text-f1-foreground`,`bg-f1-background-warning`, etc.

F0 uses TailwindCSS to apply semantic tokens. That means you need to assign a
CSS class to the component to apply a token. The classes are generated
automatically and follow the pattern _[property to set]-f1-[token name]_:

- `bg-f1-background-primary` sets `background-color` CSS property to the
  semantic token `background-primary`.
- `text-f1-foreground-success` sets `color` CSS property to the
  `foreground-success` semantic token.

## Bare colors

☝️ Avoid using bare colors directly. It is an escape hatch in cases where
semantic tokens do not fit.

Bare colors are named color values — for example, `neutral-50` or `warning-70`.
They are available as HSL triplets in a form of CSS variables (e.g.
`--neutral-50`). You will have to use `style` property in order to apply them:

```tsx
const MyButton = () => (
  <Button
    style={{ color: "hsl(var(--warning-70))" }}
    label="Send all my money"
  />
)
```

As you see, they are not convenient. This is by design. Do not use them unless
there is no other way.
