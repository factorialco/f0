# AI-Assisted Development

F0 uses a layered system of configuration files to guide AI coding assistants — agents like OpenCode and
GitHub Copilot coding agent, and chat tools like GitHub Copilot Chat. This page documents what each file
does, how they relate to each other, and the single-source-of-truth principle that keeps them from drifting.

## Why Structured AI Config?

Without explicit guidance, AI assistants guess at conventions: they invent prop patterns, import Radix
directly instead of using `@/ui/` wrappers, skip `zeroRender`, or add snapshot tests. Structured config
makes conventions machine-readable, reproducible, and enforceable — reducing review churn and keeping
generated code consistent with the codebase.

## File Hierarchy

Every AI config file in F0 has a single, non-overlapping responsibility. Rules flow **down** through
the hierarchy; files lower in the tree **reference** the layer above rather than repeating it.

## Single Source of Truth

The core principle: **every rule lives in exactly one file**. All other files that need that rule
either reference the authoritative file or remain silent on the topic.

```
Monorepo principles (TypeScript, pnpm, English)
    └── AGENTS.md (root)          ← owns this
            └── .github/copilot-instructions.md
                    "load packages/react/AGENTS.md for full conventions"

Component rules (naming, props, styling, testing, i18n, a11y)
    └── packages/react/AGENTS.md  ← owns this
            └── Skills reference it:
                "Always check packages/react/AGENTS.md first — it has the concise rules.
                 This skill provides the detailed how."

Code examples / step-by-step patterns
    └── .skills/*/references/*.md ← owns this
            loaded via the SKILL.md index
```

### What this means in practice

| You want to...                                            | Go to...                                                     |
| --------------------------------------------------------- | ------------------------------------------------------------ |
| Add a new rule about how components must be built         | `packages/react/AGENTS.md`                                   |
| Add a code example showing how to implement a rule        | The relevant `references/*.md` file inside a skill           |
| Add a new multi-step workflow (e.g. a new migration type) | Create a new `.skills/<name>/SKILL.md`                       |
| Update a rule that already exists                         | Update it in exactly one place; all other files reference it |
| Check where a rule lives                                  | Run the `f0-ai-config-hygiene` skill audit                   |

### What to avoid

- **Never copy a rule from AGENTS.md into a skill file.** If a skill needs to remind the agent of a rule, it should say _"see `packages/react/AGENTS.md` — [rule name]"_ instead of restating it.
- **Never add component rules to root `AGENTS.md`.** Root AGENTS.md is monorepo-wide; package-specific rules belong in `packages/react/AGENTS.md`.
- **Never write detailed code examples in AGENTS.md.** Keep AGENTS.md concise — it loads on every request. Long code examples belong in skill reference files, which are loaded on demand.

## The Copilot Split

Two files both serve GitHub Copilot, and that's intentional — they target different Copilot features:

| File                              | GitHub Copilot feature                                                                                                                                                                                        |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `.github/copilot-instructions.md` | **Copilot Chat** (IDE chat panel, `@workspace`) and **code completions** (inline suggestions). These are always-on features that receive a condensed context window.                                          |
| `AGENTS.md`                       | **Copilot coding agent** (the autonomous PR-creating agent). Copilot's coding agent reads the nearest `AGENTS.md` in the directory tree per the [openai/agents.md spec](https://github.com/openai/agents.md). |

Because these are separate code paths, they can't share a single file. `copilot-instructions.md` stays
minimal — it covers just enough for inline suggestions to be useful, then defers to `AGENTS.md` for
the detailed conventions.

## Skills System

Skills are on-demand instruction packages. They are **not** loaded automatically on every agent request —
they're loaded explicitly when a task matches their description.

```
.skills/
  f0-quality-gate/
    SKILL.md               — orchestrator: triggers Phase 1 commands + 4 parallel subagents
  f0-component-patterns/
    SKILL.md               — index + key rules summary
    references/
      component-architecture.md
      context-and-state.md
      styling-and-animation.md
      i18n-patterns.md
  f0-unit-testing/
    SKILL.md               — testing rules + patterns index
    references/
      unit-test-patterns.md
  f0-storybook-stories/
    SKILL.md               — story file patterns
    references/
      story-patterns.md
  f0-storybook-testing/
    SKILL.md               — play functions + axe-playwright a11y tests
    references/
      storybook-test-patterns.md
  f0-code-review/
    SKILL.md               — review checklist (blocking vs. suggestion)
  f0-experimental-component-migration/
    SKILL.md               — step-by-step migration from experimental/ to stable
  f0-pr/
    SKILL.md               — gh CLI PR workflow with correct title format
  f0-ai-config-hygiene/
    SKILL.md               — audits AI config files for instruction duplication
```

### Skill structure rules

- `SKILL.md` must be concise — it's the entry point, not the documentation.
- Detailed content goes in `references/*.md`, linked from `SKILL.md`.
- Skills **reference** `packages/react/AGENTS.md` for rules; they do not restate them.
- Skills list when to load them and when **not** to load them.

## Subagents (Quality Gate)

The `f0-quality-gate` skill uses **parallel subagents** to run four independent reviews simultaneously
after any code change. Each subagent is a focused AI agent that:

1. **Subagent A — Code Review**: Checks component architecture against `f0-code-review` conventions.
2. **Subagent B — Accessibility**: Audits interactive/animated elements using the `a11y` skill (WCAG 2.1/2.2 AA).
3. **Subagent C — Storybook Coverage**: Verifies new props have stories, new variants appear in the Snapshot story, removed props are cleaned from stories.
4. **Subagent D — Test Coverage**: Checks behavioral coverage of new props, branches, callbacks, error paths, and state transitions.

The gate runs automatically after any code modification. Skills, AGENTS.md, and docs-only changes are exempt.

### Why subagents instead of one big review?

Each domain (code quality, a11y, Storybook, testing) requires loading a different skill with its own
reference files. Loading all of them in a single context would make the review shallow. Parallel
subagents each get a focused context window and can go deep in their domain simultaneously.

## Adding or Modifying AI Configuration

### Adding a new rule

Put it in `packages/react/AGENTS.md` under the relevant section. Keep it concise — one sentence
or a short bullet. If it needs a code example, add the example to the relevant `references/*.md`
file inside the matching skill, and link to it from AGENTS.md with "See `f0-<skill>` skill for examples."

### Adding a new skill

1. Create `.skills/<name>/SKILL.md` with a frontmatter `name` and `description`.
2. Add a `## When to Use` and `## When NOT to Use` section.
3. Reference `packages/react/AGENTS.md` for rules — don't restate them.
4. Add the skill to the Skills list in `packages/react/AGENTS.md`.
5. If the skill has detailed code examples, put them in `references/*.md` and link from `SKILL.md`.

### Modifying copilot-instructions.md

Only add content that is unique to Copilot Chat / inline completions. If the content also belongs
in `AGENTS.md`, add it there and reference it from `copilot-instructions.md`.

### Auditing for duplication

Load the `f0-ai-config-hygiene` skill to run a systematic audit of all AI config files. It will
identify rules stated in multiple files and report which file should own each rule.
