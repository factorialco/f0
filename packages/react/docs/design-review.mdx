# Design Review

The design review is the **design-focused part of validation** (step 4 / DoD Phase 2) — not a separate ceremony. It happens in the [#f0-support](https://factorialteam.slack.com/archives/C082ZNKS403) thread, where a **Foundations designer** gives the OK, and the component's owners approve the PR via [`CODEOWNERS`](https://github.com/factorialco/f0/blob/main/CODEOWNERS). Code quality is checked separately by the `f0-quality-gate`; this covers the subjective, systemic calls automation can't.

**When it applies:** a new component, or a change to a **stable** component's API, visual design, or behavior. A low-impact change on an experimental component skips it — the owner iterates freely.
