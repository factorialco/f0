# Component Status

Overview of all F0 components, their API stability, and documentation coverage.

For the definition of each status level, see the [Maturity Model](/?path=/docs/components-maturity--docs) page.

## How to read this table

| Column         | Description                                                                                                                                                     |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Component**  | Component name as it appears in Storybook                                                                                                                       |
| **Zone**       | Source folder: `components` (stable public), `patterns` (composite), `sds` (satellite design systems), `kits`, `layouts`, `experimental` (legacy), `deprecated` |
| **API Status** | Derived from the `tags` array in the story file. If no status tag is set, it shows as "No tag"                                                                  |
| **Unit Tests** | Whether a `__tests__/` folder or `.test.tsx` file exists near the component                                                                                     |
| **MDX Docs**   | Whether an `.mdx` documentation file exists alongside the stories                                                                                               |

## How to update a component's status

Edit the `tags` array in the component's `.stories.tsx` file:

```tsx
const meta: Meta<typeof F0MyComponent> = {
  tags: ["autodocs", "stable"], // or "experimental", "deprecated", "internal"
}
```

The table data is regenerated by running:

```bash
node scripts/generate-component-status.mjs
```
