# Component Status

Overview of all F0 components, their API stability, and documentation coverage.

For the definition of each status level, see the [Maturity Model](/?path=/docs/lifecycle-components-maturity--docs) page.

> **Looking for a richer view?** An expanded dashboard with adoption, age, test coverage, and promotion-readiness signals may be added in the future. For now, this table is generated nightly from the codebase.

## How to read this table

| Column         | Description                                                                                                                                            |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Component**  | Component name as it appears in Storybook                                                                                                              |
| **Zone**       | Source folder: `components` (stable public), `patterns` (composite), `sds` (Domain specific), `kits`, `layouts`, `experimental`, `deprecated` |
| **API Status** | Derived from the `tags` array in the story file. If no status tag is set, it shows as "No tag"                                                         |
| **Unit Tests** | Whether a `__tests__/` folder or `.test.tsx` file exists near the component                                                                            |
| **MDX Docs**   | Whether an `.mdx` documentation file exists alongside the stories                                                                                      |

> The **Internal** badge in the API Status column refers to components in `src/ui/` (Radix wrappers used only inside F0). It is **not** a maturity level — it's a folder convention. The three maturity levels are **Experimental**, **Stable**, and **Deprecated** (see [Component Maturity](/?path=/docs/lifecycle-components-maturity--docs)).

## How to update a component's status

Edit the `tags` array in the component's `.stories.tsx` file:

```tsx
const meta: Meta<typeof F0MyComponent> = {
  tags: ["autodocs", "stable"], // or "experimental", "deprecated"
}
```

The table data is regenerated by running:

```bash
node scripts/generate-component-status.mjs
```
