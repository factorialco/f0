# Component Maturity

F0 components have **three** maturity levels. Each one tells consumers what to expect from the component's API and what guarantees it carries.

| Level            | What it guarantees                                                                            |
| ---------------- | --------------------------------------------------------------------------------------------- |
| **Experimental** | Useful enough to ship, but the API may change without a major version bump.                   |
| **Stable**       | Production-ready. Any breaking change requires a major version bump and a migration guide.    |
| **Deprecated**   | Still works, but scheduled for removal. Has `@deprecated` + `@removeIn` + `@migration` JSDoc. |

## Story tag is mandatory

Every component story **must** carry exactly one of these tags. There is no default.

| Required story tag | Maturity      |
| ------------------ | ------------- |
| `experimental`     | Experimental  |
| `stable`           | Stable        |
| `deprecated`       | Deprecated    |

The tag is the single source of truth for maturity. Tooling (components dashboard, CI gates, promotion automation) will read this tag to enforce the Definition of Done as that automation lands — see [Definition of Done](/?path=/docs/lifecycle-definition-of-done--docs).

---

## Experimental

**Where it lives:** `src/experimental/<Category>/F0Name/` or `src/sds/<area>/`.

**Story tag:** `experimental`.

**Owned by:** the team that built it.

**Guarantees:**

- The component works for its documented use cases.
- Has unit tests, a Storybook story, axe a11y check, and at least Acceptable MDX docs.

**Does NOT guarantee:**

- API stability across versions.
- Coverage of edge cases beyond the documented ones.
- Long-term maintenance if the owning team disbands.

**How it gets promoted:** see the [Definition of Done](/?path=/docs/lifecycle-definition-of-done--docs) Phase 4–5.

---

## Stable

**Where it lives:** `src/components/`, `src/patterns/`, `src/kits/`, `src/layouts/`, `src/hooks/`.

**Story tag:** `stable`.

**Owned by:** Foundations team.

**Guarantees:**

- Backwards-compatible changes only (no breaking changes without a major version bump).
- Good or Gold MDX docs.
- Full test coverage (unit + Storybook play + a11y).
- Listed in the public exports.

---

## Deprecated

**Story tag:** `deprecated`.

**Owned by:** the original owner (until removed).

**Required JSDoc tags:**

```ts
/**
 * @deprecated Use F0NewComponent instead.
 * @removeIn 2.0.0
 * @migration https://github.com/factorialco/f0/blob/main/packages/react/docs/migrations/f0-old-to-new.md
 */
```

**Removal SLA:** at least 1 quarter (≥90 days) after the `@deprecated` tag is added.

Full policy: [Deprecation & Removal](/?path=/docs/lifecycle-release-and-versioning--docs).

---

## What "Internal" means

Maturity is about API stability, nothing else. Folder location is orthogonal:

- `src/ui/` — third-party primitive wrappers (Radix, shadcn). Not part of F0's public API. Sidebar group: **UI Wrappers** (hidden in published Storybook).
- `src/components/_primitives/<Name>/` — DS primitives such as `F0InputField`. Visible to contributors under `Components/Primitives/` but not re-exported from `src/f0.ts`. JSDoc-marked `@internal`.

Neither bypasses the maturity model: a UI Wrapper or a Primitive is still tagged `experimental`, `stable`, or `deprecated` like any other component.

For the full placement rules (components, patterns, layouts, kits, Domain specific, primitives), see [Where it goes](?path=/docs/lifecycle-where-it-goes--docs).
