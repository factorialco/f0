## RichTextDisplay

`RichTextDisplay` is a read-only component designed to render rich text content safely. It supports both HTML and GitHub Flavored Markdown (GFM), including complex elements like code blocks, nested lists, and tables.

Use this component whenever you need to display content generated by `RichTextEditor` or `NotesTextEditor`, or when rendering documentation and AI-generated responses.

**Props:**
- `content`: `string` (Required) - The raw string content to be rendered.
- `format`: `'html' | 'markdown'` (Optional) - Defines how the content should be parsed. Default: `'html'`.
- `className`: `string` (Optional) - Custom CSS classes for the container.
- `allowDangerousHtml`: `boolean` (Optional) - Whether to allow non-sanitized HTML. Default: `false`.

**Usage Example:**
```tsx
import { RichTextDisplay } from './rich-text/RichTextDisplay';

const markdownContent = `
### Project Update
- [x] Initial Research
- [ ] Design Phase
| Task | Priority |
| --- | --- |
| API Integration | High |
`;

export const ContentPreview = () => {
  return (
    <RichTextDisplay 
      content={markdownContent}
      format="markdown"
      className="prose-sm"
    />
  );
};
```

**Best Practices:**
- **Markdown Support:** Use `format="markdown"` for GFM content to ensure tables and task lists render correctly.
- **Styling:** The component usually relies on a "prose" class (like Tailwind Typography) to style standard HTML elements (h1, p, li).

**Edge Cases & Gotchas:**
- **Sanitization:** By default, the component sanitizes HTML to prevent XSS. If you expect specific tags (like iframes or custom embeds), ensure they are permitted in the configuration.
- **Empty Content:** If `content` is an empty string, the component renders an empty container. Handle "No content available" states at the parent level if necessary.
