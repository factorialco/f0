# Banners

Banners are high-visibility components used to communicate important information to users. The SDS Banner system includes standard **Callouts** for status messaging and **AiBanners** specifically designed for AI-generated content and insights.

## Props

- **content** (required): `string` - HTML or plain text content displayed in the banner body.
- **title** (required): `string` - The title displayed in the banner header.
- **variant** (required): `number` - The visual variant of the callout (e.g., Info, Warning, Critical, AI).
- **actions** (optional): `Array<ActionButton>` - Array of action buttons (maximum of 2).
- **onClose** (optional): `() => void` - Callback function triggered when the close button is clicked.
- **primaryAction** (optional): `string` - Configuration for the primary action button.
- **secondaryAction** (optional): `string` - Configuration for the secondary action button.

## Sub-components

### Callout
The standard banner for system messaging. It supports several semantic variants:
- **Info**: General information (uses `InfoCircle` icon).
- **Positive**: Success messages (uses `CheckDouble` icon).
- **Warning**: Cautionary messages (uses `Warning` icon).
- **Critical**: Errors or high-priority warnings.

### AiBanner
A specialized variant featuring a gradient background, used exclusively for AI-generated insights, summaries, or suggestions.

## Usage Examples

### Standard Informational Callout
```tsx
import { Callout } from '@sds/banners';

const MyComponent = () => (
  <Callout
    title="Information Updated"
    content="Your profile changes have been saved successfully."
    variant="info"
    onClose={() => console.log('Closed')}
  />
);
```

### AI Insight Banner with Actions
```tsx
import { AiBanner } from '@sds/banners';

const AiInsight = () => (
  <AiBanner
    title="AI Suggestion"
    content="Based on Elias's team availability, this request looks good to approve."
    variant="ai"
    actions={[
      { label: 'Approve', onClick: handleApprove },
      { label: 'View Schedule', onClick: handleView }
    ]}
  />
);
```

### Critical Callout with Actions
```tsx
import { Callout } from '@sds/banners';

const ErrorState = () => (
  <Callout
    title="System Error"
    content="We were unable to process your request. Please try again."
    variant="critical"
    primaryAction="Retry"
    secondaryAction="Cancel"
  />
);
```

## Variants and States

- **Default**: Standard display with title and content.
- **Without Actions**: A simplified version for purely informational content.
- **Skeleton / Skeleton Compact**: Use these states as placeholders during data loading to prevent layout shift.
- **With Close Only**: Use when a banner is dismissible but requires no further user action.

## Best Practices

- **Limit Actions**: Do not exceed two action buttons per banner to maintain clarity.
- **Variant Selection**: Use `Critical` only for errors that require immediate attention. Use `AiBanner` only for content generated by LLMs or machine learning models.
- **Conciseness**: Keep titles short and content focused on a single message or insight.
