# F0AmountCalculator

A numeric input that supports two rendering modes: **inline** as a standard form field, or **popover** as a floating panel triggered by an icon button. Ideal for amount, percentage, or any numeric value that benefits from inline context (e.g. "of 300,00 €").

## Positioning

- Use `F0AmountCalculator` in bespoke product surfaces where you need **inline contextual amount text** (value + "of X") or **popover editing**.
- In `F0Form`, prefer regular `number` / `percentage` fields for consistency, and use `helpText` for supportive text under the input.
- Do not use `F0AmountCalculator` only to display a plain percentage input with no contextual right-side content.

## Guidelines

### Design best practices

**When to use**

- Use inline mode when the numeric field is always visible and is a primary data point in the form
- Use popover mode on dense surfaces (table cells, compact rows) where showing the input inline would break the layout
- Use `extraContent` to provide units or contextual values directly next to the input (e.g. `"of 300,00 €"`)
- Use `inputWidth` to keep the input compact and prevent it from stretching across the available width
- In `F0Form`, keep using `number` / `percentage` and put contextual explanation in `helpText`

**When not to use**

- For simple number inputs without context — use `NumberInput` directly
- In `F0Form` when you only need below-input explanatory text — use field `helpText`
- As a primary CTA button — the popover trigger is an icon button for revealing an input, not a general action trigger
- For inputs that require rich formatting (currency symbols, thousands separators configured per locale) without passing `locale` — always pass a locale

**Layout**

- In inline mode with `extraContent`, the label and messages are automatically hoisted above and below the row — no extra wrapper needed
- Keep `inputWidth` fixed when `extraContent` is present to prevent the row from jumping as the value changes
- In popover mode, the panel has a minimum width of `220px` (`min-w-[220px]`) and adapts to content

### Content best practices

- Keep `extraContent` short — a currency amount, a unit label, or a percentage base
- Label text should follow the same rules as other form fields: sentence case, imperative where possible
- Error messages should be specific: `"Value must be between 0 and 100"` not `"Invalid value"`

### Behavior

**Inline with extra content**

When `extraContent` is provided, label, hint, and error are rendered outside the flex row. The inner `NumberInput` receives `hideLabel` and suppressed hint/error so the row height stays consistent.

## Accessibility

- In popover mode the trigger button always requires a `label` prop — it is used as the accessible name and tooltip even when the icon is self-explanatory
- The Calculator icon is the default trigger; override with `popover.icon` when the context suggests a different affordance
- Error and warning messages are rendered below the input and announced via the standard `InputMessages` component
- `disabled` uses native disabled semantics — the input is removed from tab order
