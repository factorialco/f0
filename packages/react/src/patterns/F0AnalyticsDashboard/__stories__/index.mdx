# Analytics dashboard

`F0AnalyticsDashboard` renders declarative metric, chart, and collection widgets with shared filters, export controls, and editable layouts. Host applications can add widget-specific actions without putting callbacks in persisted dashboard item definitions.

### Widget actions

A `DashboardItemAction` requires a stable `id`, a user-facing `label`, and an `onClick` callback. It may also provide an `icon` and `critical` emphasis. Actions appear directly in the widget menu, separately from file downloads.

### Behavior

- WHEN `itemActions` returns actions → THEN they appear at the top level of that widget's menu, before explanation, download, and delete controls.
- WHEN an item action is selected → THEN the widget menu closes before the action callback runs, allowing the callback to open another overlay safely.
- WHEN a widget is loading or its data request fails → THEN host actions remain available so recovery workflows are not trapped behind the error.
- WHEN `itemActions` returns `undefined` or an empty array and no built-in action applies → THEN the widget menu is omitted.
- WHEN the dashboard has one item or an item enters fullscreen → THEN the same host actions remain available.

### Screen reader behavior

- Widget menu triggers have an accessible “Other actions” label.
- Loading widgets expose `aria-busy` and a polite live region.
- Action labels identify the operation; provide distinct labels when several actions could otherwise sound identical.
