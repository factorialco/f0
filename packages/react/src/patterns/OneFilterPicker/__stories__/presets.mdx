# FiltersPicker / Presets

## Introduction

The presets are predefined configurations that allow users to quickly access
frequently used filtered data views. This simplifies navigation, as there is no
need to manually adjust the filters every time you want to change the table
view.

Presets use **replace behavior**: clicking a preset replaces all current filters
with the preset's filter configuration. A preset shows as selected only when the
current filters exactly match the preset's filter value. If the user manually
adds or changes any filter, the preset becomes deselected.

To ensure an optimal user experience, it is recommended to carefully select the
presets, prioritizing those that display the most relevant and frequently
consulted information. It is essential to avoid overloading the table with too
many preconfigured filters.

## Setting Presets

To enable presets in your data collection component, supply the presets
attribute in the source prop.

```tsx
<OneFiltersPicker>
  ... presets=
  {[
    {
      label: "Preset 1",
      filter: {
        department: ["Engineering"],
      },
      itemsCount: (value) => {
        // This is a promise that returns a number, typically a fetch call
        // The value is the current filters value
        return new Promise((resolve) => {
          setTimeout(() => {
            resolve(10)
          }, 1000)
        })
      },
    },
  ]}
</OneFiltersPicker>
```

## Presets

### label

`string`

The label of the preset to display in the UI.

### filter

`FiltersState<Filters>`

The filter configuration to apply when this preset is selected. Clicking a
preset replaces all current filters with this value.

### itemsCount

`(value: FiltersState<Filters>) => number`

The items count of the preset. This is a function that returns a promise or a
value. It gets the current filters value as an argument.

> itemCount function is called on render, so it should be a pure function and
> you should cache the result.

## Preset Behavior

### Replace mode

Presets always use replace behavior:

1. **Selecting**: Clicking an unselected preset replaces all current filters with
   the preset's filter configuration
2. **Deselecting**: Clicking an already-selected preset clears all filters
3. **Selection indicator**: A preset shows as "selected" only when the current
   filter state exactly matches the preset's filter (same keys and values, no
   extra keys)
4. **Manual filter changes**: If the user manually adds, removes, or changes any
   filter, all presets become deselected because the filter state no longer
   exactly matches any preset

### Usage

```tsx
const presets = [
  {
    label: "Status: Pending",
    filter: { status: ["pending"] },
  },
  {
    label: "Status: Approved",
    filter: { status: ["approved"] },
  },
  {
    label: "Status: Paid",
    filter: { status: ["paid"] },
  },
]
```

## Result Count

When the `resultCount` prop is provided to `OneFilterPicker`, the chips row
displays a "N results for:" prefix before the filter chips, showing how many
items match the current filter selection.

In the `OneDataCollection` component, this is automatically wired to the total
items count from the data source.
