# Map

An interactive, on-brand vector map for putting records on a map - the
spatial sibling of `F0Graph`. Themed to f0 (light and dark) over keyless
OpenFreeMap tiles, with markers, clustering, selection, controls, and a
built-in accessible list alternative.

## Quickstart

Pass `markers`; each is a coordinate plus a semantic variant. `F0Map` owns the
camera, theming, clustering, label collision, and selection.

```tsx
import { F0Map } from "@factorialco/f0-react"

const markers = [
  { id: "hq", coordinates: [2.19, 41.4], variant: "workplace", label: "HQ" },
  {
    id: "ada",
    coordinates: [2.17, 41.38],
    variant: "employee",
    firstName: "Ada",
    lastName: "Lovelace",
    label: "Ada Lovelace",
  },
]

export function Example() {
  return <F0Map markers={markers} />
}
```

## Overview

### Purpose

- Put records **in space** - offices, workplaces, people, sites - with one
  consistent, on-brand map instead of a per-team Google Maps rebuild.
- Let users **scan and locate** at a glance: brand-colored pins over a muted
  basemap, with clustering and label collision keeping dense areas legible.
- Provide the **accessible, themeable** map the product lacked: light/dark, a
  keyboard/screen-reader list alternative, and no per-request tile bill.

`F0Map` is not a static image or a single-pin preview. Reach for it when the
**spatial relationship** between records matters.

### Anatomy

<Controls of={F0MapStories.WithMarkers} />

A Map is composed of:

1. **Basemap** - the f0-themed vector tiles (light/dark), muted so pins lead.
2. **Markers** - DOM pins from your `markers`, one per record, by variant.
3. **Clusters** - nearby markers gathered into a pile that expands on click.
4. **Controls** - locate, fit-to-markers, zoom in / out (bottom-left).
5. **List** - a synchronized, screen-reader-operable list of every marker,
   which also becomes the visible fallback if the map can't render.

## Variants

Markers are semantic - pick a `variant` and pass its data; appearance (color,
size, label placement, selection) stays uniform across the map.

- **`workplace`** - a building glyph on a fixed brand hue (all sites match).
- **`employee`** - a person avatar (its own identity color).
- **`company`** - a company avatar.
- **`default`** - a plain colored pin.
- **`current_location`** - the "you are here" dot (see below).

Selecting a marker grows it to a pinned, lifted state; neighbors it would cover
fold to a dot until it's deselected.

## Behaviors

### Clustering

Clustering is **automatic** - it's how the map behaves, not something you turn
on. Nearby markers gather as you zoom out and separate as you zoom in; there is
no `cluster` prop to think about. Lone markers only merge when nearly touching,
but once a cluster forms it absorbs its neighbors, so dense pockets collapse
into one pile instead of a ring of stray pins. Clicking a cluster flies to its
bounds (every member stays in view). See the `WithClusters` and `WorldClusters`
stories.

### Selection & highlight

`selectedMarkerId` / `onMarkerSelect` drive controlled single selection (the
grown pin). `highlightedId` is a separate, lighter channel - it floats a marker
above the rest, keeps its label, and flies to it - meant for an **external
search / reveal** (search lives outside the map, as in F0Graph).

### Current location

`showCurrentLocation` renders a "you are here" dot once the user has granted
location; the locate control requests permission on demand and drops the dot.
See the `CurrentLocation` story.

### Routes & arcs

`routes` and `arcs` draw lines on the map as GL layers beneath the markers.

- **`routes`** take a `coordinates` array and draw a polyline through it exactly
  as given. `F0Map` renders the path; it does **not** compute routing - fetch
  that from your own engine/server (e.g. OSRM) and pass the vertices. See the
  `Routes` story.
- **`arcs`** take a `from` / `to` pair and `F0Map` computes the curved
  connection between them (the flight-path look). See the `Arcs` story.

Both share styling: a palette `variant` (or an escape-hatch `color`), `width`,
`opacity`, and `dashed`. Providing `onRouteClick` / `onArcClick` enables hover
highlight (the line widens) and click. Lines are folded into the auto-fit, so
`fitToMarkers` and the Fit control frame them alongside the markers.

### Fit, framing & gestures

The camera auto-fits the markers on load (unless an `initialViewport` is given).
`gestureHandling` defaults to `"cooperative"`: a plain scroll scrolls the page
and Ctrl/⌘ + wheel (or two fingers) zooms, so an embedded map never traps the
page scroll. Use `"greedy"` for a dedicated/full-screen map.

### Presentation

`fullScreen={false}` (default) frames the map as a card - rounded corners and a
subtle border, controls inset 16px. `fullScreen` drops the frame for an
edge-to-edge map. Theme follows the nearest `.dark` ancestor unless `theme` is
forced.

`projection="globe"` renders the world as a 3D sphere at low zoom and eases
into a flat mercator view as you zoom in - best for a world-scale view (drag to
spin). It re-projects live if the prop changes. See the `Globe` story.

## Guidelines

### When to use

<DoDonts
  do={{
    description: "Use Map when you need to:",
    guidelines: [
      "Place records geographically - offices, workplaces, sites, people - so their spatial relationship is visible",
      "Let users scan, cluster, and locate across many points at once",
      "Offer one consistent, on-brand, accessible map across the product",
      "Show where something is as part of a larger record view",
    ],
  }}
  dont={{
    description: "Don't use Map when:",
    guidelines: [
      "A single fixed location would read better as a static preview image",
      "The data has no coordinates - geocoding is out of scope (done server-side)",
      "A list or table conveys the same information with less interaction cost",
      "You need routing computed for you, freehand drawing, or heatmaps - out of scope (you can pass a pre-computed route to draw)",
    ],
  }}
/>

### Content best practices

- **Labels are scannable.** Keep them to a place or person name - 2-4 words, no
  sentences. Dense areas drop labels automatically; don't rely on them at low
  zoom.
- **Use the right variant.** `workplace` for sites, `employee`/`company` for
  people/orgs - the fixed colors keep categories readable.
- **Let clustering work.** Dense areas gather into piles on their own; pass the
  full marker set and let the map manage overlap rather than fighting it by hand.
- **Tolerate missing coordinates.** Skip position-less records; the map never
  assumes every record is mappable.

### Accessibility

- The **synchronized list is the text alternative**, not decoration: a WebGL
  canvas is opaque to assistive tech, so screen-reader and keyboard users
  operate the map through the list (activating an item selects + flies to it).
- A **skip-to-list** link jumps past the canvas; an `aria-live` region
  announces the marker count.
- If WebGL is unavailable, the list becomes the **visible fallback**; a
  tile-load failure shows a retry banner.
- All control and ARIA strings come from the `map.*` i18n namespace - override
  per-instance via `F0MapControls` `labels`.

### Performance

- `maplibre-gl` is heavy (~268KB) and needs a WebGL context - avoid mounting
  many maps on one page.
- Pass the full marker array up front (the map needs every coordinate to
  cluster and fit); it does not paginate.

## Props

All public props on `<F0Map>`, grouped by behavior.

### Data

| Prop           | Type                   | Default | Description                                      |
| -------------- | ---------------------- | ------- | ------------------------------------------------ |
| `markers`      | `F0MapPoint[]`         | `[]`    | Points to render, one per record.                |
| `routes`       | `F0MapRoute[]`         | `[]`    | Polylines drawn through their coordinates as-is. |
| `arcs`         | `F0MapArc[]`           | `[]`    | Curved connections between two coordinates.      |
| `onRouteClick` | `(id: string) => void` | —       | Route click; presence enables hover + click.     |
| `onArcClick`   | `(id: string) => void` | —       | Arc click; presence enables hover + click.       |

### Selection

| Prop                      | Type                           | Default | Description                                       |
| ------------------------- | ------------------------------ | ------- | ------------------------------------------------- |
| `selectedMarkerId`        | `string \| null`               | —       | Controlled selected marker.                       |
| `defaultSelectedMarkerId` | `string \| null`               | `null`  | Uncontrolled initial selection.                   |
| `onMarkerSelect`          | `(id: string \| null) => void` | —       | Fired on marker click or background click.        |
| `highlightedId`           | `string \| null`               | `null`  | Emphasise + fly to a marker without selecting it. |

### Viewport

Clustering is automatic (no prop) - see [Clustering](#clustering).

| Prop              | Type            | Default             | Description                            |
| ----------------- | --------------- | ------------------- | -------------------------------------- |
| `fitToMarkers`    | `boolean`       | `true` w/o viewport | Frame all markers on load.             |
| `initialViewport` | `F0MapViewport` | city-level          | Initial `{ center, zoom }`, read once. |
| `minZoom`         | `number`        | fitted              | Farthest zoom out.                     |
| `maxZoom`         | `number`        | `18`                | Closest zoom in.                       |

### Presentation & interaction

| Prop                  | Type                          | Default         | Description                                                 |
| --------------------- | ----------------------------- | --------------- | ----------------------------------------------------------- |
| `theme`               | `"auto" \| "light" \| "dark"` | `"auto"`        | Force a theme; `auto` follows the nearest `.dark` ancestor. |
| `mapStyle`            | `F0MapStylePair`              | f0 styles       | Light/dark style pair override.                             |
| `interactive`         | `boolean`                     | `true`          | Allow pan/zoom.                                             |
| `gestureHandling`     | `"cooperative" \| "greedy"`   | `"cooperative"` | Cooperative requires Ctrl/⌘ + wheel to zoom (embed-safe).   |
| `fullScreen`          | `boolean`                     | `false`         | `false` frames as a card; `true` is edge-to-edge.           |
| `projection`          | `"mercator" \| "globe"`       | `"mercator"`    | `"globe"` renders the world as a 3D sphere.                 |
| `showControls`        | `boolean`                     | `true`          | Show the locate / fit / zoom toolbar (when interactive).    |
| `showCurrentLocation` | `boolean`                     | `false`         | Show the "you are here" dot when location is granted.       |
| `loading`             | `boolean`                     | `false`         | Show the skeleton instead of the map.                       |
| `ariaLabel`           | `string`                      | `"Map"` (i18n)  | Accessible name for the map region.                         |

### Imperative handle (`ref`)

| Method             | Description                               |
| ------------------ | ----------------------------------------- |
| `getMap()`         | The raw MapLibre instance (escape hatch). |
| `focusMarker(id)`  | Center on a marker and select it.         |
| `fitToMarkers()`   | Frame all markers.                        |
| `clearSelection()` | Clear the current selection.              |
