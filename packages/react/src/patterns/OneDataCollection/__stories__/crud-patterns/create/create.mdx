# Create pattern

Create is the entry point for adding a new resource from a Data Collection. The
collection toolbar should expose the action, and the chosen container should
match the amount of guidance, structure, and time the user needs to complete
the task.

## When to use

- Use a Default Dialog when the form is short, focused, and can be completed in
  one pass.
- Use a Wizard Dialog when creation requires multiple steps, staged decisions,
  or review before submission.
- Use Kanban Lane Create when the new resource should start in a specific lane
  or workflow state. **Recommended** for scoped Kanban creation.
- Use Editable Table add-row actions for lightweight rows that can be created
  inline. **Recommended** only when the row does not need a full setup form.
- Keep create actions at collection level instead of attaching them to a
  specific row.
- Preserve the collection context so users can return to the list after
  creation.

## Guidance

Use one clear collection primary action for creation. Label it with the user
goal, not the implementation detail. Keep the dialog title aligned with the
action label so users immediately understand they are in the correct flow.

When a wizard is needed, split the flow into steps that match a real decision
model. Each step should have a distinct purpose. Do not turn a short single-form
create flow into a wizard only to make it feel more structured.

For table-heavy contexts, a Right Dialog can work as an alternative
when people need to preserve list context while creating. Keep this as a table
variant, not the default create pattern.

For Kanban, lane create is **Recommended** when the lane is meaningful input:
the user is not only creating a resource, they are creating it in a known state.
For Editable Table, add-row is **Recommended** only for lightweight records that
can be created with defaults and refined inline.
