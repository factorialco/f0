# Data collection / CRUD patterns

## Introduction

CRUD patterns describe the recommended UX for creating, opening, updating, and
deleting records from a Data Collection. They are product patterns that consumers
wire through collection actions, item actions, item click handlers, bulk actions,
dialogs, wizards, or routes.

Data Collection does not automatically choose a dialog or page for you. Use these
patterns to choose the right container for the resource and task.

## Create and update guidance

Use a default dialog for standard create and update flows. Update should mirror
create whenever possible: reuse the same form structure, field order, validation
rules, and terminology when users edit the same information.

Complex resources can use lightweight creation followed by richer update/details
completion inside the resource. In lightweight create flows, use explicit CTAs:

- <code>Save and close</code> creates the item and returns to the collection.
- <code>Save and open resource</code> creates the item and opens the resource
  page for completion.

Choose the primary CTA based on the expected next step. Use Save and close as
primary for fast or repeated creation. Use Save and open resource as primary when
the resource is incomplete after creation.
