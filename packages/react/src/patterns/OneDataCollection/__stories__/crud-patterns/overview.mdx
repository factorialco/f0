# Data collection / CRUD patterns

## Introduction

CRUD patterns describe the recommended UX for creating, opening, updating, and
deleting records from a Data Collection. They are product patterns that consumers
wire through collection actions, item actions, item click handlers, bulk actions,
dialogs, wizards, or routes.

Data Collection does not automatically choose a dialog or page for you. Use these
patterns to choose the right container for the resource and task.

These stories are reference compositions, not visual mockups. Each scenario is
built with the same F0 layouts, headers, dialogs, wizards, and collection
components that production flows are expected to use.

## Scenario stories

## Create, read, and update guidance

Use a default dialog for standard create, read, and update flows. Update should
mirror create whenever possible: reuse the same form structure, field order,
validation rules, and terminology when users edit the same information.

Use the default dialog as the baseline read pattern when the user needs a quick
inspection flow. Move to a page with breadcrumbs when the resource becomes
complex enough to need persistent navigation, deeper configuration, or a longer
editing surface.
