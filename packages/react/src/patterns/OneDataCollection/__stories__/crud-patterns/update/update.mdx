# Update pattern

Update patterns define how a user updates an existing resource from a Data
Collection. The update flow should usually mirror create so users do not need to
relearn the form, field order, or terminology when updating the same data.

## When to use

- Use a default dialog when updating fits in a focused overlay.
- Use a right-position dialog only for Table and List views when keeping the
  collection visible improves comparison or navigation between items.
- Use bulk update when the current visualization provides explicit
  multi-selection and users can review the affected items before confirming.
- Use Card click for read first and expose Edit at the top of the read dialog.
  **Recommended** when card click is already the read affordance.
- Use Kanban drag/move for state updates and Kanban bulk update when selected
  cards make scope explicit. **Recommended** for board workflows.
- Use Editable Table inline cell editing for visible field-level updates.
  **Recommended** when the user is changing values already shown in the table.
- Reuse the same form structure as create whenever the user updates the same
  information.
- Avoid inventing a separate update-only layout without a strong UX reason.

## Guidance

Update should feel like create with existing values prefilled. Reusing the same
form structure reduces cognitive load, improves predictability, and keeps
validation and terminology aligned.

The right-position dialog is an exception, not the default. Use it when the
collection itself is part of the update workflow, such as reviewing adjacent
rows or moving through items while preserving context. Fall back to the default
dialog below medium breakpoints.

Bulk update follows the same selection pattern as bulk delete. Use it only when
the visualization makes selection scope explicit, such as checkbox-based
selection, so users can clearly review which items are included before they
confirm the change.

Checkbox-based selection is the clearest current example, but the rule is the
explicitness of the selected scope, not the Table visualization itself. Kanban
bulk update is **Recommended** when selected cards remain visible across lanes
and the action bar confirms the affected scope.

Card update should preserve the default read-first behavior: clicking the card
opens the read dialog, and the read dialog exposes Edit at the top. Editable
Table update should use inline cell editing for visible field-level changes and
reserve dialogs for full-resource updates.
