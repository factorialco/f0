# AI chat textarea

Provides the complete message composer for an AI conversation, including
attachments, mentions, dictation, contextual chips, welcome content, and
host-owned compact toolbar controls.

### Content best practices

- Keep placeholders short and action-oriented so they remain useful as the
  available width changes.
- Use concise labels for toolbar controls; explanatory copy belongs outside the
  action row.
- Keep disclaimers factual and link only to information that clarifies the AI
  interaction.
- Write welcome suggestions as recognizable tasks, with the detailed request in
  the prompt rather than the visible title.

**Correct:** “Ask about your team’s time off.”

**Incorrect:** “Please enter your query here to get started.”

<DoDonts
  do={{
    description: "Use direct, specific composer copy.",

  dont={{
    description: "Avoid generic instructions and explanatory labels.",
    guidelines: [

### Usage examples

**File attachments**

Use the attachment configuration to validate and upload files before the
structured submit payload is emitted.

**Compact host control**

Use <code>toolbarStart</code> for a small mode or scope selector that must sit
inside the composer action row.

**Clarifying question**

Use <code>clarifyingUI</code> when the conversation temporarily requires a
structured answer instead of free text.

### Behavior

- WHEN <code>toolbarStart</code> is supplied in the normal composer → THEN it renders after the attachment action and before F0-owned trailing actions.
- WHEN an injected control is activated → THEN click propagation stops at the slot boundary so the form does not move focus back to the textarea.
- WHEN <code>clarifyingUI</code> replaces the input → THEN the normal action row and <code>toolbarStart</code> are absent.
- WHEN audio recording replaces the normal action row → THEN <code>toolbarStart</code> is absent; it returns while transcription uses the normal row.
- WHEN submission is accepted → THEN <code>onSubmit</code> receives text, files, context, and quote as separate structured fields.

## Accessibility

The composer preserves document order: attachments and <code>toolbarStart</code>
come before microphone and send controls. Injected controls keep their own
focus and must provide their own accessible names, keyboard behavior, and
disabled state.
