# F0DataChart

Unified chart component. Uses a discriminated `type` prop to select the chart variant.

```tsx
import { F0DataChart } from "@factorialco/f0-react"
```

## Quick Reference

| Type      | Best for                             | Key props                                                  |
| --------- | ------------------------------------ | ---------------------------------------------------------- |
| `bar`     | Comparing categories or time periods | `series`, `categories`, `orientation`, `stacked`           |
| `line`    | Trends over time                     | `series`, `categories`, `lineType`, `showArea`, `showDots` |
| `funnel`  | Conversion pipelines, hiring stages  | `series`, `sort`, `orient`, `showConversion`, `colorScale` |
| `pie`     | Part-of-whole distributions          | `series`, `innerRadius`, `showPercentage`                  |
| `radar`   | Multi-dimensional comparisons        | `series`, `indicators`, `showArea`                         |
| `gauge`   | Single KPI / progress indicator      | `value`, `min`, `max`, `name`                              |
| `heatmap` | Density across two dimensions        | `data`, `xCategories`, `yCategories`, `showLabels`         |

All variants share an optional `emptyState` prop — see [Empty states](#empty-states).

## Empty states

When data is empty (empty `series`, empty `data`, all-zero values, or a missing gauge `value`), `F0DataChart` auto-renders a faded chart skeleton with a centered message so you never see a bare axis. No prop is required — it is the default behavior.

Default copy is **"No data available"** / **"Try a different date or fewer filters"**.

<Canvas of={EmptyStatesStories.EmptyBar} />

Use the `emptyState` prop to customize copy, replace the UI entirely, or skip detection:

```ts
type F0DataChartEmptyStateProps = {
  title?: string
  description?: string
  /** Render-prop escape hatch — replaces the entire UI. */
  render?: () => ReactNode
  /** Skip empty detection and render the chart as usual. */
  disabled?: boolean
}
```

## Line Chart

Visualize trends over time. Supports linear, smooth, and step interpolation, area fill, dashed lines, and data point dots. Single-series charts get a soft area gradient under the line; multi-series charts always render lines only because overlapping fills become unreadable. The chart is responsive: at narrow widths the legend and axes are progressively hidden.

## Customization

### Value & Category Formatters

All chart types support `valueFormatter` to customize how numbers are displayed in axes, labels, and tooltips. Bar and line charts also support `categoryFormatter` for axis labels.

```tsx
<F0DataChart
  type="bar"
  valueFormatter={(v) => `${v / 1_000_000}M €`}
  categoryFormatter={(v) => v.slice(0, 3)}
  // ...
/>
```

### ECharts Escape Hatch

Bar and line chart types accept an `echartsOptions` prop for advanced customization. Options are shallow-merged on top of the generated config.

```tsx
<F0DataChart
  type="bar"
  echartsOptions={{
    animation: false,
    grid: { left: 80 },
  }}
  // ...
/>
```

### Color Tokens

Use F0 design tokens for custom colors: `"malibu"`, `"purple"`, `"grass"`, `"orange"`, `"red"`, `"viridian"`, `"barbie"`, `"lilac"`, `"smoke"`, `"army"`, `"flubber"`, `"indigo"`, `"camel"`, `"radical"`, `"yellow"`.
