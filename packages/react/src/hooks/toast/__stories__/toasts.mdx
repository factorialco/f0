# Toasts

## Introduction

### Definition

`toasts` is an imperative API for triggering toast notifications. Its methods are **not hooks** — you can call them from anywhere (event handlers, async callbacks, even outside React), as long as your app is wrapped in `<F0Provider>` (which mounts the `ToastProvider`). Open toasts live in a module-level store that the provider renders into a portal.

### Purpose

- **No hook, no wiring:** Call `toasts.open(...)` directly — no hook to call, no `useState`, no conditional rendering
- **Automatic lifecycle:** Toasts auto-dismiss after a configurable duration and clean up after themselves
- **Consistent UX:** Standardized toast patterns with stacking, hover-to-expand, and smooth animations
- **Flexible API:** Support for variants, descriptions, action buttons/links, custom durations, and persistent toasts
- **Programmatic control:** Dismiss individual toasts (`toasts.close`) or all at once (`toasts.closeAll`) from anywhere in your code

The provider handles rendering, positioning, stacking, timers, and cleanup automatically.

## Basic Usage

### Simple toast

The minimum required option is `title`:

```tsx
import { toasts } from "@factorialco/f0-react"

function MyComponent() {
  return (
    <button onClick={() => toasts.open({ title: "Profile updated" })}>
      Save Profile
    </button>
  )
}
```

### Toast with description

Use `description` to provide additional context below the title:

```tsx
toasts.open({
  title: "File uploaded",
  description: "report-q4-2024.pdf has been uploaded successfully.",
})
```

## Variants

Toasts support four visual variants to communicate intent. When you don't pass a `variant`, the `"default"` variant is used — so there's no need to set it explicitly.

### Default

```tsx
toasts.open({ title: "Changes saved" })
```

### Success

```tsx
toasts.open({ variant: "success", title: "Payment processed" })
```

### Warning

```tsx
toasts.open({ variant: "warning", title: "Storage almost full" })
```

### Error

```tsx
toasts.open({ variant: "error", title: "Upload failed" })
```

## Actions

Toasts can include up to two actions: one button and one link. Actions are dismissed automatically when clicked, unless `keepOpen` is set.

### Button action

Use a button action for in-page operations:

```tsx
toasts.open({
  title: "Message deleted",
  actions: {
    type: "button",
    label: "Undo",
    onClick: () => restoreMessage(),
  },
})
```

### Link action

Use a link action to navigate the user to a related page:

```tsx
toasts.open({
  title: "Report ready",
  variant: "success",
  actions: {
    type: "link",
    label: "View report",
    href: "/reports/latest",
  },
})
```

### Button + link (combined)

You can combine one button and one link:

```tsx
toasts.open({
  title: "Export complete",
  variant: "success",
  actions: [
    {
      type: "button",
      label: "Undo",
      onClick: () => cancelExport(),
    },
    {
      type: "link",
      label: "Download",
      href: "/exports/file.csv",
    },
  ],
})
```

## Duration and Persistence

### Default duration

Toasts auto-dismiss after **10 seconds** by default.

### Custom duration

Override the duration (in milliseconds) via the `duration` option:

```tsx
// Dismiss after 3 seconds
toasts.open({ title: "Copied to clipboard", duration: 3000 })

// Dismiss after 30 seconds
toasts.open({ title: "Background sync started", duration: 30000 })
```

### Persistent toasts

Set `persistent: true` to keep a toast visible until the user manually closes it or you dismiss it programmatically. When `persistent` is set, `duration` is ignored.

```tsx
const id = toasts.open({
  title: "Uploading…",
  description: "Your file is being uploaded. Please wait.",
  persistent: true,
})

// Dismiss it later when done
await uploadFile()
toasts.close(id)
```

## Programmatic Control

### Returning the toast ID

`toasts.open()` returns the ID of the created toast, which you can use to remove it later:

```tsx
const id = toasts.open({ title: "Saving…", persistent: true })

await save()

toasts.close(id)
```

### Custom ID

Provide your own `id` to make referencing the toast easier — for example, to avoid duplicate toasts for the same resource:

```tsx
toasts.open({
  id: "sync-status",
  title: "Syncing…",
  persistent: true,
})

// Calling with the same id again replaces the existing toast
toasts.open({
  id: "sync-status",
  title: "Sync complete",
  variant: "success",
})
```

### toasts.close

Dismiss a specific toast by ID:

```tsx
import { toasts } from "@factorialco/f0-react"

const id = toasts.open({ title: "Processing…", persistent: true })

// Later...
toasts.close(id)
```

### toasts.closeAll

Dismiss all currently visible toasts at once:

```tsx
import { toasts } from "@factorialco/f0-react"

// e.g. on logout
toasts.closeAll()
```

## Type Reference

### toasts

```tsx
const toasts: {
  /**
   * Show a toast. Returns the id of the created toast.
   */
  open(options: ToastOptions): ToastId

  /**
   * Dismiss a toast by id.
   */
  close(id: ToastId): void

  /**
   * Dismiss all currently visible toasts.
   */
  closeAll(): void
}
```

### ToastOptions

Options passed to `toasts.open()`.

```tsx
type ToastOptions = {
  /**
   * The title of the toast.
   */
  title?: string

  /**
   * Optional description displayed below the title.
   */
  description?: string

  /**
   * The visual variant of the toast.
   * @default "default"
   */
  variant?: F0ToastVariant

  /**
   * The actions to display in the toast.
   * Supports a single button, a single link, or a [button, link] tuple.
   */
  actions?:
    | F0ToastAction
    | [F0ToastAction]
    | [ToastActionButton, ToastActionLink]

  /**
   * Optional id for the toast. If not provided, a random id will be generated.
   * Providing the same id twice replaces the existing toast.
   */
  id?: ToastId
} & (
  | {
      /**
       * The duration of the toast in milliseconds.
       * @default 10000
       */
      duration?: number
      persistent?: undefined
    }
  | {
      duration?: undefined
      /**
       * If true, the toast will not auto-dismiss.
       * The user must close it manually, or call toasts.close(id) / toasts.closeAll().
       */
      persistent: true
    }
)
```

### F0ToastVariant

```tsx
type F0ToastVariant = "default" | "success" | "warning" | "error"
```

### F0ToastAction

Action types supported by the `actions` option.

```tsx
type ToastActionButton = {
  type: "button"
  label: string
  onClick: () => void
  icon?: IconType
  /**
   * If true, the toast will not close when this action is clicked.
   */
  keepOpen?: boolean
}

type ToastActionLink = {
  type: "link"
  label: string
  href: string
  /**
   * If true, the toast will not close when this action is clicked.
   */
  keepOpen?: boolean
}

type F0ToastAction = ToastActionButton | ToastActionLink
```
