# useToast Hook

## Introduction

### Definition

The `useToast` hook provides an imperative API for triggering toast notifications in React applications. It wraps the `ToastProvider` context internally, handling all state management and rendering, so you can trigger toasts from anywhere in your component tree with a single function call.

### Purpose

- **No state management:** Trigger toasts imperatively — no `useState`, no conditional rendering
- **Automatic lifecycle:** Toasts auto-dismiss after a configurable duration and clean up after themselves
- **Consistent UX:** Standardized toast patterns with stacking, hover-to-expand, and smooth animations
- **Flexible API:** Support for variants, descriptions, action buttons/links, custom durations, and persistent toasts
- **Programmatic control:** Dismiss individual toasts or all at once from anywhere in your code

The hook handles rendering, positioning, stacking, timers, and cleanup automatically.


## Basic Usage

### Simple toast

The minimum required option is `title`:

```tsx
import { useToast } from "@/hooks/toast"

function MyComponent() {
  const { toast } = useToast()

  return (
    <button onClick={() => toast({ title: "Profile updated" })}>
      Save Profile
    </button>
  )
}
```

### Toast with description

Use `description` to provide additional context below the title:

```tsx
toast({
  title: "File uploaded",
  description: "report-q4-2024.pdf has been uploaded successfully.",
})
```

## Variants

Toasts support four visual variants to communicate intent. The default variant is `"default"`.

### Default

```tsx
toast({ variant: "default", title: "Changes saved" })
```

### Success

```tsx
toast({ variant: "success", title: "Payment processed" })
```

### Warning

```tsx
toast({ variant: "warning", title: "Storage almost full" })
```

### Error

```tsx
toast({ variant: "error", title: "Upload failed" })
```

## Actions

Toasts can include up to two actions: one button and one link. Actions are dismissed automatically when clicked, unless `keepOpen` is set.

### Button action

Use a button action for in-page operations:

```tsx
toast({
  title: "Message deleted",
  variant: "default",
  actions: {
    type: "button",
    label: "Undo",
    onClick: () => restoreMessage(),
  },
})
```

### Link action

Use a link action to navigate the user to a related page:

```tsx
toast({
  title: "Report ready",
  variant: "success",
  actions: {
    type: "link",
    label: "View report",
    href: "/reports/latest",
  },
})
```

### Button + link (combined)

You can combine one button and one link:

```tsx
toast({
  title: "Export complete",
  variant: "success",
  actions: [
    {
      type: "button",
      label: "Undo",
      onClick: () => cancelExport(),
    },
    {
      type: "link",
      label: "Download",
      href: "/exports/file.csv",
    },
  ],
})
```

## Duration and Persistence

### Default duration

Toasts auto-dismiss after **10 seconds** by default.

### Custom duration

Override the duration (in milliseconds) via the `duration` option:

```tsx
// Dismiss after 3 seconds
toast({ title: "Copied to clipboard", duration: 3000 })

// Dismiss after 30 seconds
toast({ title: "Background sync started", duration: 30000 })
```

### Persistent toasts

Set `persistent: true` to keep a toast visible until the user manually closes it or you dismiss it programmatically. When `persistent` is set, `duration` is ignored.

```tsx
const { toast } = useToast()

const id = toast({
  title: "Uploading…",
  description: "Your file is being uploaded. Please wait.",
  variant: "default",
  persistent: true,
})

// Dismiss it later when done
await uploadFile()
removeToast(id)
```

## Programmatic Control

### Returning the toast ID

`toast()` returns the ID of the created toast, which you can use to remove it later:

```tsx
const { toast, removeToast } = useToast()

const id = toast({ title: "Saving…", persistent: true })

await save()

removeToast(id)
```

### Custom ID

Provide your own `id` to make referencing the toast easier — for example, to avoid duplicate toasts for the same resource:

```tsx
toast({
  id: "sync-status",
  title: "Syncing…",
  persistent: true,
})

// Calling with the same id again replaces the existing toast
toast({
  id: "sync-status",
  title: "Sync complete",
  variant: "success",
})
```

### removeToast

Dismiss a specific toast by ID:

```tsx
const { toast, removeToast } = useToast()

const id = toast({ title: "Processing…", persistent: true })

// Later...
removeToast(id)
```

### clearAll

Dismiss all currently visible toasts at once:

```tsx
const { clearAll } = useToast()

// e.g. on logout
clearAll()
```

## Type Reference

### ToastOptions

Options passed to the `toast()` function.

```tsx
type ToastOptions = {
  /**
   * The title of the toast.
   */
  title?: string

  /**
   * Optional description displayed below the title.
   */
  description?: string

  /**
   * The visual variant of the toast.
   * @default "default"
   */
  variant?: F0ToastVariant

  /**
   * The actions to display in the toast.
   * Supports a single button, a single link, or a [button, link] tuple.
   */
  actions?:
    | F0ToastAction
    | [F0ToastAction]
    | [ToastActionButton, ToastActionLink]

  /**
   * Optional id for the toast. If not provided, a random id will be generated.
   * Providing the same id twice replaces the existing toast.
   */
  id?: ToastId
} & (
  | {
      /**
       * The duration of the toast in milliseconds.
       * @default 10000
       */
      duration?: number
      persistent?: undefined
    }
  | {
      duration?: undefined
      /**
       * If true, the toast will not auto-dismiss.
       * The user must close it manually, or call removeToast(id) / clearAll().
       */
      persistent: true
    }
)
```

### UseToastReturn

The object returned by `useToast()`.

```tsx
type UseToastReturn = {
  /**
   * Trigger a new toast notification.
   * @returns The id of the created toast.
   */
  toast: (options: ToastOptions) => ToastId

  /**
   * Remove a toast by id.
   */
  removeToast: (id: ToastId) => void

  /**
   * Remove all currently visible toasts.
   */
  clearAll: () => void
}
```

### F0ToastVariant

```tsx
type F0ToastVariant = "default" | "success" | "warning" | "error"
```

### F0ToastAction

Action types supported by the `actions` option.

```tsx
type ToastActionButton = {
  type: "button"
  label: string
  onClick: () => void
  icon?: IconType
  /**
   * If true, the toast will not close when this action is clicked.
   */
  keepOpen?: boolean
}

type ToastActionLink = {
  type: "link"
  label: string
  href: string
  /**
   * If true, the toast will not close when this action is clicked.
   */
  keepOpen?: boolean
}

type F0ToastAction = ToastActionButton | ToastActionLink
```
