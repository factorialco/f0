# Alert

Keeps users aware of something relevant to their current context — a missing
setup, a failed process, a completed action — without interrupting the workflow.
For transient feedback use a toast; for decisions that block the workflow use
`F0Dialog`.

### Content best practices

- **Title**: Sentence case, no period. State the situation clearly ("Company page inactive")
- **Description**: One or two sentences. Explain what the situation means and what can be done about it
- **Action label**: Verb-first, no articles ("Request info", "Activate now")
- **Link label**: Descriptive of the destination ("See all invoices"), never generic ("Click here")

### Behavior

- At narrow container widths the layout stacks vertically: icon and text on top, action and link below
- At wider containers the layout is horizontal: icon and text on the left, action and link on the right
- The alert stretches to fill its container — control width from the parent
- When `action.disabled` is true, the button remains visible but non-interactive — keep it visible so the action remains discoverable

## Usage

### Narrow layout

At small container widths the layout stacks vertically — useful for sidebars,
panels, or any tight container.

<Canvas of={Stories.Narrow} />

```tsx
<F0Alert
  variant="warning"
  title="Company page inactive"
  description="You are missing the company page. It is mandatory to be activated."
  action={{ label: "Request info", onClick: () => {} }}
/>
```

### Disabled action

Use when the action is conditionally unavailable. Keep the button visible so
the action is discoverable even when blocked.

<Canvas of={Stories.DeactivatedAction} />

```tsx
<F0Alert
  variant="warning"
  title="Company page inactive"
  description="You are missing the company page. It is mandatory to be activated."
  action={{ label: "Request info", onClick: () => {}, disabled: true }}
/>
```
