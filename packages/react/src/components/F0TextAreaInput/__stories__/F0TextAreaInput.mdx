# F0TextAreaInput

`F0TextAreaInput` is the writable multi-line text field — a box where the user types longer text spanning multiple lines (notes, descriptions, comments, feedback). It is a thin wrapper over [F0InputField](?path=/docs/components-primitives-f0inputfield--documentation) that swaps the single-line `<input>` for a `<textarea>` and adds height auto-grow.

## Guidelines

### Design best practices

#### When to use

- Free-form text that spans more than one line: notes, descriptions, comments, feedback, addresses.
- Whenever the user might press Enter to start a new paragraph.

#### When NOT to use

- The value is single-line free-form text. Use [F0TextInput](?path=/docs/components-inputs-text-input--documentation).
- The text is rich (formatting, mentions, links). Use the dedicated rich-text editor.

## Behavior

What `F0TextAreaInput` adds on top of F0InputField.

### Auto-grow

The textarea grows in height as the user types, never forcing the user to scroll inside a tiny box. It starts at `rows` lines tall (defaults to 2) and expands until it hits `maxHeight` (in pixels), at which point an inner scrollbar appears.

- `rows={4}` → starts 4 lines tall, grows from there.
- `maxHeight={300}` → grows up to 300px, then scrolls.
- `maxHeight` unset → grows without bound.

### New lines

`Enter` inserts a new line — there is no `onPressEnter` shortcut, because submitting a multi-line field with `Enter` would conflict with paragraph breaks. Wrap in `<form onSubmit>` and use a submit button.
