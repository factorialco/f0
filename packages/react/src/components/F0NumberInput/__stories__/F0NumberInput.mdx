# F0NumberInput

`F0NumberInput` is the writable numeric field for forms — a single-line input where the user types a number, with locale-aware formatting, optional min/max clamping, an optional stepper, and an optional units suffix. It builds on [F0InputField](?path=/docs/components-primitives-f0inputfield--documentation) and adds numeric value model, locale handling, decimals, min/max clamping, stepper and units.

---

## Overview

### Anatomy

<Canvas of={Stories.Primary} />

- **Input area**: where the user types. Optionally accompanied by a stepper and/or a units suffix.
- **Units suffix** _(optional)_: read-only label rendered at the end of the input (e.g. `EUR`, `%`, `kg`) when `units` is set.
- **Stepper arrows** _(optional)_: increment/decrement buttons rendered when `step` is set; respect `min`/`max`.
- **Label, hint, status message**: standard input chrome.

## Guidelines

### Design best practices

#### When to use

- The user enters a quantity: amounts, counts, percentages, ages, ratings.
- The value has a clear numeric meaning and may need locale formatting.
- The valid range is bounded (`min`/`max`) and you want the field to enforce it.

#### When NOT to use

- The value is free-form text (names, codes, IDs that may include letters). Use [F0TextInput](?path=/docs/components-inputs-text-input--documentation).
- The value is a duration in hours/minutes. Use [F0DurationInput](?path=/docs/components-inputs-duration-input--documentation).
- The value is one of a small, discrete set of numbers (e.g. a 1–5 rating). Use a selector with the available options instead.
- The value is an identifier that happens to look numeric (phone, postal code, credit card). These are formatted strings, not quantities — leading zeros, spaces and country variants matter, and locale-aware formatting would corrupt them. Use [F0TextInput](?path=/docs/components-inputs-text-input--documentation) with the right `autocomplete` and `inputMode`.

## Behavior

### Value model

The value is `number | null`. `null` means the field is empty.

- `value={42}` → input shows the formatted number for the current `locale`.
- `value={null}` → input is empty.
- `onChange` fires with `number | null`. Consumers should treat `null` as "user cleared the field".

### Input mode

`F0NumberInput` renders a native `<input type="text">` with `inputMode="decimal"` (or `"numeric"` when `maxDecimals={0}`), so mobile keyboards show the right keypad without the pitfalls of `type="number"` — no stripped leading zeros, no broken locale separators, and no browser-native steppers competing with the F0 stepper.

### Locale

`locale` is required. The decimal separator follows the locale:

- `locale="en-US"` → `1234.5`
- `locale="es-ES"` → `1234,5`
- `locale="pt-BR"` → `1234,5`

Grouping (thousand separators) is intentionally **disabled** while editing — typing `1234` should not jump to `1,234` and break the cursor.

### Decimals

`maxDecimals` caps the number of decimal digits the user can type. Invalid keystrokes are blocked at `onBeforeInput` (the character never appears).

- `maxDecimals={0}` → digits only; `.` and `,` are blocked; `inputMode="numeric"`.
- `maxDecimals={2}` → up to two decimals; further decimal keystrokes are blocked.
- Unset → unlimited decimals.

### Min and max

If the parsed value falls outside `[min, max]`, it is clamped to the boundary before `onChange` fires.

- `min={0}` with the user typing `-5` → `onChange(0)`.
- `max={100}` with the user typing `150` → `onChange(100)`.

When clamping is a no-op (the value is already in range), the **raw input is preserved** — typing `"17."` stays as `"17."` instead of collapsing to `"17"`, so the user can keep typing decimals.

### Auto hint

When `min` and/or `max` are set and `hint` is not explicitly provided, F0NumberInput renders a localized hint:

- `min` + `max` → `"Between {min} and {max}"`.
- `min` only → `"Greater than {min}"`.
- `max` only → `"Less than {max}"`.

Pass an explicit `hint` to override.

### Stepper

When `step` is set, up/down arrow buttons render inside the input.

<Canvas of={Stories.WithStep} />

- Clicking up/down adds/subtracts `step` from the current value.
- If the result would cross `min`/`max`, the step is a no-op.
- If `value` is `null`, the first click seeds the field with `step` (not `0`).

### Units

`units` renders a tag at the end of the input. It is decorative — it does not change the emitted number.
