# F0AudioPlayer

Plays back a single audio recording with play/pause, a draggable seek bar, and a
current/total time label. `F0AudioPlayer` is the bare controls bar;
`F0AudioPlayerCard` wraps it in a titled card with a kebab menu for playback
speed and custom actions. This component is playback only — to capture live
audio use `useAudioRecorder` with `RecordingWaveform` instead.

### Content best practices

- **Title**: Sentence case, no period. Name what the recording is, not its format
  - **Correct:** "AI call with Alex Williams", "Screening call — Backend role"
  - **Incorrect:** "Recording", "audio_final_v2.mp3"
- **Subtitle**: Use it for the date and time, or other at-a-glance metadata
- **Action labels**: Verb-first and specific ("Download recording", "Copy link"), never generic ("Click here")

### Behavior

- **No built-in loading or error UI.** The player tracks `error` and loading
  internally but does not render anything for them yet. Supply a valid `src` and
  handle failures through the `onError` callback (e.g. show your own message or
  hide the player). A broken source otherwise shows a player stuck at `0:00`.
- **Controlled or uncontrolled playback.** Leave `playing` unset to let the
  component manage play state, or pass `playing` + `onPlayingChange` to drive it
  from the parent (e.g. to pause one recording when another starts).
- **Seeking commits on release.** While dragging, the playhead follows the thumb
  visually; `onSeek` fires (and `audio.currentTime` updates) only when the user
  releases. Seeking is disabled until the duration is known.
- **Speed menu lives on the card.** `playbackRates` (default `1×, 1.5×, 2×`) is
  exposed through `F0AudioPlayerCard`'s kebab. The bare `F0AudioPlayer` has no
  speed control. Pass `playbackRates={[]}` to show only the `actions` in the
  kebab.

### Screen reader behavior

- The player is a `role="group"` labelled "Audio player"; override it per
  instance with `ariaLabel`.
- The play/pause button's accessible name flips between "Play" and "Pause" with
  the state.
- The seek bar is a `role="slider"` with `aria-valuetext` announcing the
  position as "M:SS of M:SS".
- The kebab trigger is labelled "Recording options". The active playback speed
  is conveyed visually with a check icon (the options use standard menu
  semantics, not radio semantics).
- Icons are decorative — the labels, title, and time text carry the meaning.
