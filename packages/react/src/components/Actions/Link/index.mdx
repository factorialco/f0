# Link

## Introduction

### Definition

The Link component is a versatile navigation element that provides clickable
text or content that navigates users to different pages, sections, or external
resources within the application

### Purpose

- Enable navigation between different pages and sections
- Provide clear visual indicators for clickable content
- Support both internal and external navigation patterns

## Guidelines

### Content best practices

- Use descriptive link text that clearly indicates the destination
- Avoid generic phrases
- Keep link text concise but informative (2-5 words)

### Design best practices

#### When to use

- **Navigation**: Use links for navigating between pages, sections, or external
  resources
- **References**: Use links to reference related content or documentation

#### When not to use

- **Actions**: Use buttons for actions that perform operations (submit, delete,
  etc.)
- **Primary actions**: Use buttons for the main call-to-action on a page
