# ButtonDropdown

Consolidates multiple related actions under a single button. Comes in two modes: **split** (default) promotes the most common action to a one-click trigger while keeping the rest reachable via a chevron; **dropdown** presents all actions equally under a single trigger button. For a single action use `F0Button` instead.

### Design best practices

#### When to use

Use **split mode** when one action is clearly more frequent than the rest — the user can trigger it in one click without opening the menu.

Use **dropdown mode** when all actions are equally likely and none should be promoted — a single trigger opens the full list.

### Content best practices

- Item labels: verb-first, sentence case, no period ("Save draft", "Delete record")
- Group labels: short noun phrases that categorize the items below ("File actions", "Export options")
- Trigger label (dropdown mode): a noun or verb phrase describing the category, not a specific item ("Actions", "Export")
- Descriptions: one short sentence under 60 characters that clarifies what the action does

### Behavior

- In split mode the first item is selected by default. Clicking the main button triggers it directly; the chevron opens the menu with all remaining items.
- In dropdown mode all items appear in the menu. The `trigger` prop sets an independent label for the trigger button.
- The `value` prop (split mode) controls which item is currently selected. When omitted, defaults to the first item's value.
- `disabled` and `loading` apply to the entire control — both the main button and the chevron are affected.

### Usage examples

**Items with descriptions and a critical action**

Use `description` to clarify what each item does when the label alone is ambiguous. Mark destructive actions with `critical: true` — they render in a danger style to signal irreversibility.

**Grouped items**

Use `ButtonDropdownGroup` objects to organize items into labeled sections with visual separators. Useful when the list mixes actions of different categories.

**Dropdown mode with custom trigger label**

Set `trigger` to give the button a label that describes the category rather than a specific item. Use this when the item list may change or when no single item should dominate the label.

**With tooltip**

### Screen reader behavior

- The main button (split mode) has `aria-label` set to the selected item's label
- The chevron button (split mode) has a visually hidden label (`sr-only`) using the `actions.more` i18n string
- Item icons are decorative — meaning is carried by the label and optional description
- Critical items are distinguished by color only — no additional ARIA role marks them as destructive
- The dropdown is Radix-based — it traps focus while open and restores it to the trigger on close
