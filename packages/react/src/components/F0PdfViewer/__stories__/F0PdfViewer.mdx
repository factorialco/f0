# F0PdfViewer

An experimental, read-only document viewer. Its default (and namesake) mode
renders PDFs through [`react-pdf`](https://github.com/wojtekmaj/react-pdf)
(which wraps `pdf.js`) as a continuous, scrollable list of pages. Via the
`kind` prop it also previews **spreadsheets** (xlsx/xls/csv), **Word**
documents (docx) and **text/markdown** files — every kind carries a toolbar
with its own controls, and hosts can append custom actions to it.

## Import

```tsx
import { F0PdfViewer } from "@factorialco/f0-react"
;<F0PdfViewer url="/path/to/document.pdf" filename="document.pdf" />
```

The component fills its container, so give it a sized wrapper (height + width).

## Sizing

`F0PdfViewer` is `height: 100%` / `width: 100%` — it has no intrinsic size and
takes the height of its parent. Always render it inside a container with a
defined height, otherwise it collapses.

```tsx
// Fixed height
<div className="h-[640px]">
  <F0PdfViewer url={url} filename="doc.pdf" />
</div>

// Fill a flex layout (use min-h-0 so the inner scroll works)
<div className="flex h-full flex-col">
  <Header />
  <div className="min-h-0 flex-1">
    <F0PdfViewer url={url} filename="doc.pdf" />
  </div>
</div>
```

## Document kinds

`kind` selects the document family. It defaults to `"pdf"`, which keeps the
full pdf.js viewer unchanged. The other kinds render read-only previews, each
lazy-loaded in its own chunk (SheetJS, docx-preview and the markdown pipeline
never load until a document of that kind renders). PDF-only props (`page`,
`initialScale`, `rotatable`, print…) are ignored for them.

```tsx
<F0PdfViewer url="/report.xlsx" kind="sheet" filename="report.xlsx" />
```

Every kind has a toolbar with its own peculiarities:

| Kind            | Content                                                                                        | Toolbar                                                          |
| --------------- | ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| `pdf` (default) | Continuous pages via pdf.js                                                                    | Page nav, zoom, scale selector, optional rotate, print, download |
| `sheet`         | Excel-style grid (letter columns, numbered rows), capped at 1000×100 with a truncation note    | Sheet switcher (ToggleGroup), zoom, download                     |
| `docx`          | Word page layout via [`docx-preview`](https://github.com/VolodymyrBaydalka/docxjs)             | Zoom, download                                                   |
| `text`          | `.md` renders as a sanitized document (GFM tables included); txt/log/json as monospaced source | Zoom, download                                                   |

No kind shows a title in the toolbar — same as the PDF toolbar.

If a document can't load, the pane shows a failure message but the toolbar
stays — download (and any custom action) remains reachable.

### Spreadsheets (`kind="sheet"`)

Parses xlsx, legacy xls and csv. The toolbar carries a ToggleGroup with one
toggle per sheet of the workbook to switch between them.

<Canvas of={Stories.SpreadsheetKind} />

### Word (`kind="docx"`)

Faithful page layout (breaks, tables, images) rendered client-side. Binary
`.doc` is not supported.

<Canvas of={Stories.WordKind} />

### Text and markdown (`kind="text"`)

Markdown files render as a document — headings, lists, GFM tables, code —
sanitized with DOMPurify (scripts, event handlers and images are stripped).
Anything else shows as monospaced source. Pass `mimeType` so markdown detection
doesn't rely on the filename alone.

## Custom toolbar actions

`actions` appends host-provided buttons after the built-in controls, in every
kind. The typical case is a Close button when the viewer fills a fullscreen
overlay — this is exactly how the F0Chat document preview mounts it.

```tsx
import { Cross } from "@factorialco/f0-react/icons/app"
;<F0PdfViewer
  url={url}
  kind="sheet"
  filename="report.xlsx"
  actions={[{ icon: Cross, label: "Close", onClick: onClose }]}
/>
```

## Worker setup

`pdf.js` renders in a Web Worker. F0 configures the worker automatically,
matched to the bundled `pdfjs-dist` version, so **no setup is required** in most
apps. For strict CSP or offline environments where you self-host the worker,
override it once at startup:

```tsx
import { configurePdfWorker } from "@factorialco/f0-react"

configurePdfWorker("/static/pdf.worker.min.js")
```

## Zoom

`initialScale` accepts `"page-width"` (default), `"page-fit"`, or a fixed zoom
(`"0.5"` … `"4"`). The toolbar exposes the same options plus zoom in/out steps.
The non-PDF kinds carry the same zoom controls with the fixed steps (no
page-width/page-fit — there's no page geometry to fit to); `initialScale`
itself is PDF only.

<Canvas of={Stories.PageFit} />

## Rotation

Set `rotatable` to add a toolbar control that rotates the document in 90°
steps (PDF only). Rotation is uncontrolled: the viewer keeps its own state, but
you can seed it with `initialRotation` and react to changes via
`onRotationChange` (for example, to persist it per file). Persistence is left
to the consumer so the component stays free of storage concerns.

```tsx
<F0PdfViewer
  url={url}
  filename="doc.pdf"
  rotatable
  initialRotation={getStoredRotation(fileId)}
  onRotationChange={(rotation) => storeRotation(fileId, rotation)}
/>
```

## Accessibility

- Every kind renders a <code>role="toolbar"</code> where each control is an
  icon-only button with an accessible label (page nav, zoom, print, download,
  custom actions). The sheet switcher is a single-select ToggleGroup — its
  toggles expose <code>role="radio"</code> semantics.
- The current page indicator (<code>page X / Y</code>) is announced via an
  <code>aria-live="polite"</code> region (PDF).
- Links inside PDF documents open in a new tab with
  <code>rel="noopener noreferrer"</code>; markdown HTML is sanitized with
  DOMPurify before rendering.
- The loading and skeleton states expose <code>role="status"</code> with
  <code>aria-busy</code> / <code>aria-live</code>.
