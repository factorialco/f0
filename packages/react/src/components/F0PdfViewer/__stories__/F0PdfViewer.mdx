# F0PdfViewer

An experimental, read-only PDF viewer built on
[`react-pdf`](https://github.com/wojtekmaj/react-pdf) (which wraps `pdf.js`).
It renders the document as a continuous, scrollable list of pages with a toolbar
for page navigation, zooming, printing, and downloading.

<Controls of={Stories.Default} />

## Import

```tsx
import { F0PdfViewer } from "@factorialco/f0-react"
;<F0PdfViewer url="/path/to/document.pdf" filename="document.pdf" />
```

The component fills its container, so give it a sized wrapper (height + width).

## Sizing

`F0PdfViewer` is `height: 100%` / `width: 100%` — it has no intrinsic size and
takes the height of its parent. Always render it inside a container with a
defined height, otherwise it collapses.

```tsx
// Fixed height
<div className="h-[640px]">
  <F0PdfViewer url={url} filename="doc.pdf" />
</div>

// Fill a flex layout (use min-h-0 so the inner scroll works)
<div className="flex h-full flex-col">
  <Header />
  <div className="min-h-0 flex-1">
    <F0PdfViewer url={url} filename="doc.pdf" />
  </div>
</div>
```

## Worker setup

`pdf.js` renders in a Web Worker. F0 configures the worker automatically,
matched to the bundled `pdfjs-dist` version, so **no setup is required** in most
apps. For strict CSP or offline environments where you self-host the worker,
override it once at startup:

```tsx
import { configurePdfWorker } from "@factorialco/f0-react"

configurePdfWorker("/static/pdf.worker.min.js")
```

## Zoom

`initialScale` accepts `"page-width"` (default), `"page-fit"`, or a fixed zoom
(`"0.5"` … `"4"`). The toolbar exposes the same options plus zoom in/out steps.

## Loading

While the document loads (and as a `F0PdfViewer.Skeleton` placeholder) the

## Accessibility

- The toolbar uses <code>role="toolbar"</code>; every control is an icon-only
  button with an accessible label (previous/next page, zoom in/out, print,
  download).
- The current page indicator (<code>page X / Y</code>) is announced via an
  <code>aria-live="polite"</code> region.
- Links inside the document open in a new tab with
  <code>rel="noopener noreferrer"</code>.
- The loading and skeleton states expose <code>role="status"</code> with
  <code>aria-busy</code> / <code>aria-live</code>.
