# Link

## Introduction

### Definition

The Link component is a versatile navigation element that provides clickable
text or content that navigates users to different pages, sections, or external
resources within the application

### Purpose

- Enable navigation between different pages and sections
- Provide clear visual indicators for clickable content
- Support both internal and external navigation patterns

### Mention

Use `variant="mention"` to render an inline reference to an entity (person,
team, project, …) inside running text. Mentions render as a subtle gray chip
that stays visually consistent across the design system, the rich text editor
and any other surface that produces serialized HTML.

<Canvas of={LinkStories.Mention} />

#### Rendering mentions outside a React tree

Some surfaces (Tiptap-based editors, server-rendered chat messages, sanitized
HTML payloads) need to emit mention chips as raw HTML strings, without going
through React. In those cases, import the shared visual recipe instead of
mounting `F0Link`:

```ts
import { mentionFocusableClasses } from "@/lib/recipes"

const html = `<a href="/users/jane" data-type="mention" class="${mentionFocusableClasses}">@Noor Rahimi</a>`
```

The recipe guarantees the same chip appearance and focus treatment as
`F0Link variant="mention"`. Always keep mentions semantically as `<a href>`
so screen readers and keyboard navigation work.

## Guidelines

### Content best practices

- Use descriptive link text that clearly indicates the destination
- Avoid generic phrases
- Keep link text concise but informative (2-5 words)

### Design best practices

#### When to use

- **Navigation**: Use links for navigating between pages, sections, or external
  resources
- **References**: Use links to reference related content or documentation

#### When not to use

- **Actions**: Use buttons for actions that perform operations (submit, delete,
  etc.)
- **Primary actions**: Use buttons for the main call-to-action on a page
