# F0Slider

A slider input for selecting a numeric value from a range. Supports labels, hints, status messages, tooltips with multiple visibility modes, and keyboard navigation.

---

# Overview

## Purpose

- Allows users to select a numeric value from a bounded range with a draggable thumb
- Provides continuous value adjustment with optional tooltip showing the current value
- Supports keyboard-based value adjustment following the native slider ARIA pattern
- Communicates validation status through color-coded track and range fill
- Adapts to reduced motion preferences for accessibility

# Guidelines

## Design Best Practices

### When to use

- Use a slider when users benefit from seeing relative position within a range (e.g., brightness, volume, budget proximity)
- Use a slider when the exact value is secondary to the approximate position
- Use a slider when providing continuous visual feedback during adjustment

### When NOT to use

- Do not use a slider when users need to enter a precise known value: Use a numeric input instead
- Do not use a slider for discrete options with fewer than 5 steps: Use radio buttons or a select instead
- Do not use a slider for unbounded ranges or when the range is unknown at render time

<DoDonts

    description:
      "Use a slider for adjusting relative values like brightness or volume where visual position matters",
  }}
  dont={{
    description:
      "Use a slider for entering a specific number like an employee ID or a known salary figure",
  }}
/>

## Content Best Practices

- Keep labels concise: one or two words describing the value being adjusted
- Use `minLabel` and `maxLabel` when the range boundaries are meaningful (e.g., "€0" / "€10k")
- Use `formatValue` to display values in the tooltip with appropriate units and locale formatting
- Hint text should explain the effect of the slider, not describe the slider itself

## Layout

- Slider width is determined by the parent container; use a container with a fixed or percentage width
- When using range labels, they align to the left and right edges of the track
- The tooltip appears centered above the thumb, offset with a gap
- Messages appear below the track with the same spacing as other form fields

## Accessibility

- The slider thumb has an implicit <code>role="slider"</code> with <code>aria-valuemin</code>, <code>aria-valuemax</code>, and <code>aria-valuenow</code>
- The thumb exposes its accessible name via <code>aria-label</code> (resolved from <code>label</code> or <code>ariaLabel</code>), and the visible label is associated through <code>htmlFor</code>
- The formatted value is announced to assistive tech via <code>aria-valuetext</code>, so screen readers read "50%" or "€5,000" rather than the raw number
- Keyboard navigation follows the native slider pattern: Arrow Left/Right to decrement/increment
- Validation messages are linked via <code>aria-describedby</code> and announced through a polite <code>aria-live</code> region
- The thumb hover animation respects <code>prefers-reduced-motion</code>
- The disabled state uses <code>aria-disabled</code> and removes pointer events

# Examples

## Basic Usage

<Canvas of={Stories.Default} />

```tsx
import { useState } from "react"
import { F0Slider } from "@factorialco/f0-react"

function BrightnessControl() {
  const [brightness, setBrightness] = useState(50)

  return (
    <F0Slider
      label="Brightness"
      min={0}
      max={100}
      value={brightness}
      onChange={setBrightness}
    />
  )
}
```

## With Range Labels and Formatted Values

<Canvas of={Stories.WithRangeLabels} />

```tsx
function BudgetSlider() {
  const [budget, setBudget] = useState(5000)

  return (
    <F0Slider
      label="Budget"
      min={0}
      max={10000}
      value={budget}
      onChange={setBudget}
      minLabel="€0"
      maxLabel="€10k"
      formatValue={(v) => `€${v.toLocaleString()}`}
    />
  )
}
```

<Canvas of={Stories.TooltipAlways} />

```tsx
import { useState } from "react"

function OpacityControl() {

  return (
    <F0Slider

      min={0}
      max={1}
      step={0.01}
      value={opacity}

    />
  )
}
```
