# F0Accordion

A collapsible container that organizes related content into independent sections. Each item expands and collapses on its own and can host contextual actions in its header.

---

# Overview

## Purpose

- Reveal dense content on demand, reducing visual noise and cognitive load.
- Let users decide which sections they need to inspect.
- Group related blocks of information without forcing a fixed reading order.
- Provide a slot for per-item actions (segmented controls, dropdown menus) without nesting interactive controls.

# Guidelines

## Design Best Practices

### When to use

- Long pages where related content can be grouped into self-contained sections (settings, profile details, FAQs).
- Forms split into optional steps the user can review independently.
- Lists of records where each entry has secondary detail the user may want to inspect.

### When NOT to use

- The page has only one or two short sections. Use a card or plain section instead.
- The content is critical for the task and must always be visible. Do not hide it behind a collapse.
- The component is meant for navigation. Use Tabs or a sidebar instead.

### How to use

- Allow multiple items open at the same time. Each item is independent and not mutually exclusive.
- Use `defaultOpen` on individual items, or `defaultValue` / `value` on the root component, to control which sections start expanded.
- Keep the header free of nested buttons. Use the `actions` slot for in-row controls.

## Content Best Practices

- Keep titles short and descriptive so the user can scan headers without expanding them.
- Avoid repeating the title inside the body content.
- Do not nest accordions inside accordions.

<DoDonts

  }}

## Layout

- The component fills 100% of its parent width.
- Open and close animations respect `prefers-reduced-motion`.
- Header height is fixed; only the content area grows on expand.

## Accessibility

- Each trigger is a native button with `aria-expanded` reflecting the open state.
- Only the chevron is interactive in the header, avoiding nested buttons when actions are present.
- Animations honour the user's reduced-motion preference.

<Canvas of={Stories.Controlled} />

