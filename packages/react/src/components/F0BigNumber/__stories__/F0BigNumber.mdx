# BigNumber

Highlights a single, high-priority metric — revenue, headcount, an attendance
rate — as a large formatted value with an optional label and a comparison
against a previous period. Use it as the headline figure in dashboards and
summary cards. For a metric plotted over time use `F0DataChart`; for a compact
status indicator use a tag or badge.

### Content best practices

- **Label**: Sentence case, no trailing period. Name the metric, not the calculation ("Monthly revenue", not "Sum of paid invoices this month")
- **Label length**: Keep it to a few words so the value stays the focal point
- **Comparison hint**: State the reference period plainly
  - **Correct:** "vs last month", "vs Q3", "vs last year"
  - **Incorrect:** "compared to the previous reporting period before adjustments"
- **Units**: Provide units through the numeric `value` object (e.g. currency, percentage) rather than appending them to the label

### Behavior

- `value` and `comparison` accept either a raw `number` or a numeric object (`Numeric` / `NumberWithFormatter`). Values are formatted at up to 2 decimal places using the locale from the `l10n` provider, so the component must render inside that provider.
- The comparison tag (`F0TagBalance`) renders only when a `comparison` value resolves to a finite number.
- `trend` computes `(value − comparison) / comparison × 100`. The percentage is omitted when either side is missing or the comparison is zero, so no division-by-zero or misleading percentage is shown.
- By default an increase is styled positive and a decrease negative. Pass `trend={{ invertStatus: true }}` for metrics where a lower number is better (e.g. absence rate, error count) so the coloring matches the desired direction.
- `F0BigNumber.Skeleton` renders a loading placeholder that mirrors the label and value layout — use it while data is in flight.

## Accessibility

- The trend tag conveys direction through both the signed percentage and color. Color alone is not the only cue, so the change remains understandable without color perception.
- When a lower value is the good outcome, set `invertStatus` so the coloring does not contradict the meaning — otherwise an improvement is shown in the "negative" color.
- Provide a `label` (or a clear heading in the surrounding context) so the figure is not announced as a bare number with no meaning.
