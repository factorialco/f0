# F0TextInput

The canonical writable text field — a single-line box where text, emails, passwords or other free-form values are typed. A thin wrapper over [F0InputField](?path=/docs/components-primitives-f0inputfield--documentation) that adds text-specific behavior: automatic password masking, a `"private"` mode for sensitive non-credential data, and an `onPressEnter` shortcut. For numeric data use [F0NumberInput](?path=/docs/components-inputs-number-input--documentation); for durations use [F0DurationInput](?path=/docs/components-inputs-duration-input--documentation); for search queries use [F0SearchInput](?path=/docs/components-inputs-search-input--documentation).

### Content best practices

- **Label**: Sentence case, no period. Name the value concretely ("Social security number", not "Number")
- **Placeholder**: An example of the expected format, never a substitute for the label
- **Hint**: One short sentence clarifying the format or constraint when the label is not enough

### Behavior

The `type` prop accepts every native HTML input type **except `"number"`**, which is blocked at the type level — for numeric data use `F0NumberInput`. Common values are `"text"` (default), `"email"`, `"password"`, `"url"`, and `"tel"`. It also accepts the F0-specific `"private"`. `type="file"` is accepted but provides no drag-and-drop, validation or preview — use a dedicated upload component for files.

**Password mode**

When `type="password"`:

- A lock icon replaces any `icon` prop.
- A trailing eye-toggle button appears, with i18n labels ("Show password" / "Hide password"), to flip between masked and visible text.
- The `buttonToggle` prop becomes private — consumers cannot override the eye toggle while in password mode.

**Private mode**

`type="private"` is an F0-specific subtype for **sensitive, non-credential data** — Social Security numbers, national IDs, account numbers, tokens. It masks the value like `password`, but differs in two ways that matter:

- **No lock icon** — the consumer `icon` prop is respected (default: none), so the field does not read as a login credential.
- **Password managers are disabled** — the field sets `autocomplete="off"` plus `data-1p-ignore`, `data-lpignore`, `data-bwignore` and `data-form-type="other"`, so 1Password, LastPass, Bitwarden and browser autofill never offer to save or fill it.

A trailing eye-toggle flips between masked and visible text; its accessible name is built from the field label (e.g. "Show social security number") so screen-reader users can tell multiple private fields apart. As with password mode, the `buttonToggle` prop is private. Use `type="password"` for the user's own login password, where autofill is helpful; use `type="private"` for secrets a password manager must never capture.

**Enter key**

`onPressEnter` fires when `Enter` is pressed while the input has focus. Useful for form-less search boxes, inline editing, or single-field submit flows. It does not call `preventDefault` — wrap the input in a `<form onSubmit>` if native form submission is also needed.
