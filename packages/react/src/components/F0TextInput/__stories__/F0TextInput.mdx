# F0TextInput

`F0TextInput` is the canonical writable text field — a single-line input where the user types text, passwords or emails. It is a thin wrapper over [F0InputField](?path=/docs/components-primitives-f0inputfield--documentation) that adds two text-specific behaviors: automatic password mode and an `onPressEnter` shortcut.

## Guidelines

### Design best practices

#### When to use

- Names, titles, descriptions and other free-form single-line text.
- Email addresses, URLs, usernames, codes (e.g. invoice numbers, IDs that may include letters).
- Passwords (use `type="password"` — see Behavior > Password mode).
- Identifiers that look numeric but must preserve formatting: phone numbers, postal codes, credit cards. Pair with the right `autocomplete` and `inputMode`.

#### When NOT to use

- The value is a quantity. Use [F0NumberInput](?path=/docs/components-inputs-number-input--documentation).
- The value is a duration in hours/minutes. Use [F0DurationInput](?path=/docs/components-inputs-duration-input--documentation).
- The value is a search query. Use [F0SearchInput](?path=/docs/components-inputs-search-input--documentation).
- The value is multi-line. Use F0TextAreaInput.

## Behavior

### Type

`type` accepts every native HTML input type **except `"number"`**, which is intentionally blocked at the type level — for numeric data use `F0NumberInput`. Common values: `"text"` (default), `"email"`, `"password"`, `"url"`, `"tel"`.

`type="file"` is technically accepted but does not provide drag-and-drop, validation or preview — use the dedicated upload component for file uploads.

### Password mode

When `type="password"`:

- A lock icon replaces any `icon` prop.
- A trailing eye-toggle button appears, with i18n labels ("Show password" / "Hide password"), to flip between masked and visible text.
- The `buttonToggle` prop becomes private — consumers cannot override the eye toggle while in password mode.

<Canvas of={Stories.Password} />

### Enter key

`onPressEnter` fires when the user presses `Enter` while focused on the input. Useful for form-less search boxes, inline editing or single-field submit flows. It does not preventDefault — wrap in `<form onSubmit>` if you also want native form submission.
